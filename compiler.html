Morpher: Best-in-Class CGRA Compiler

As PACE is a software-defined hardware accelerator, it requires a scalable compiler to generate high-quality mapping of applications specified in high-level programming languages such as C/C++ onto the CGRA accelerator. We have made tremendous progress in developing state-of-the-art compiler for CGRA accelerators. Our Morpher compiler is the best in class in the world in generating quality mapping, achieving fast compilation time, offering flexibility to explore different architectural designs, and providing end-to-end design flow including architecture specification, customized compiler generation, hardware synthesis, emulation, and verification/validation.

Morpher is a powerful, integrated compilation and simulation framework that can assist design space exploration and application-level developments of CGRA based systems. Morpher can take an application with a compute intensive kernel as input, compile the kernel onto a user-provided CGRA architecture, and automatically validate the compiled kernels through cycle-accurate simulation using test data extracted from the application.

Framework Capabilities

  • Real-World Applications: Morpher can handle real-world application kernels without being limited to simple toy kernels through its feature-rich compiler
  • Flexible Architecture Support: Architecture description language lets users easily specify architectural features such as complex interconnects, multi-hop routing, and memory organizations
  • Open-Source Since Dec 2022: Various research groups across the world have reached out for collaborations on this toolchain
  • Complete Toolchain: DFG generation, SPM data layout, test data extraction, CGRA mapper, mapping configurations, C++ simulation, Verilog RTL, FPGA emulation

PANORAMA: Hierarchical Mapping for Complex Loop Kernels

PANORAMA: Hierarchical Mapping Approach

We have developed two hierarchical mapping approaches to improve the compilation time. The first is PANORAMA, a divide and conquer approach for mapping complex loop kernels on CGRA. Mapping complex loop kernels onto the CGRA becomes challenging mainly due to the irregular data dependencies in complex loop kernels and limited routing resources on the CGRA fabric.

PANORAMA is a fast, scalable and portable compiler based on a divide-and-conquer approach to generate quality mapping for complex Dataflow Graphs (DFGs) onto larger CGRA. The higher-level mapping partitions the dataflow graph onto the CGRA clusters and guides the lower-level mapping, reducing overall complexity.

Technical Innovation

  • Hierarchical Decomposition: Complex DFG partitioned into clusters that can be mapped independently, then composed together
  • Cluster Dependency Graph: Higher-level abstraction guides placement of clusters onto CGRA regions
  • Lower-Level Optimization: Each cluster mapped optimally onto assigned CGRA sub-space
  • Handles Irregular Applications: Specifically designed for kernels with large, irregular DFGs that challenge conventional approaches

HiMap: Systolic Array Abstraction

While PANORAMA handles complex loop kernels, HiMap is adept at producing close to optimal solutions for regular computational kernels prevalent in existing and emerging application domains on larger CGRAs. The state-of-the-art compiler technology falls short in generating high-performance mapping within an acceptable compilation time, especially with increasing CGRA size (beyond 4×4 PE array).

The key strategy behind HiMap's efficiency and scalability is to exploit the regularity in the computation by employing a virtual systolic array as an intermediate abstraction layer in a hierarchical mapping. HiMap first maps the loop iterations of the kernel onto a virtual systolic array and then distills out the unique patterns in the mapping. These unique patterns are subsequently mapped onto sub-spaces of the physical CGRA. They are arranged together according to the systolic array mapping to create a complete mapping of the kernel.

Virtual Systolic Array Strategy

  • Intermediate Abstraction: Virtual systolic array serves as intermediate representation between loop iterations and physical CGRA
  • Pattern Extraction: Identifies repeating patterns in systolic mapping, reducing complexity from mapping entire kernel to mapping unique patterns
  • Sub-Space Mapping: Each unique pattern mapped onto CGRA sub-space independently
  • Composition: Final mapping assembled by replicating and arranging pattern mappings according to systolic structure
  • Near-Optimal Results: Can hit the performance envelope of the CGRA for regular kernels

LISA Framework for Portable and Quality Compilation on CGRAs

LISA: Portable Compilation Framework

While PANORAMA and HiMap employ hierarchical mapping to improve the mapping quality and significantly reduce the compilation time, we still need a good-quality low-level mapper/compiler. Currently, the mappers/compilers like SPR are handcrafted for CGRAs, which is challenging especially when we develop heterogeneous CGRA accelerator with techniques like REVAMP.

Therefore, we designed a portable compilation framework, called LISA, that can be tuned automatically to generate quality mapping for varied spatial accelerators. Our key contribution is to automatically identify the impact of the dataflow graph (DFG) structure characteristics (representing an application) on the mapping for a new accelerator. Towards this end, we abstract the DFG structure in graph attributes, use Graph Neural Network (GNN) to analyze the graph attributes, and identify the mapping impact for an accelerator architecture with an all-encompassing global view.

GNN-Based Approach

  • Graph Attributes Extraction: DFG structure abstracted into comprehensive graph attributes capturing structural properties
  • GNN Analysis: Graph Neural Network trained to analyze how graph attributes impact mapping quality for specific accelerator architectures
  • Label-Aware Simulated Annealing: Augments SA-based mapping approach to account for DFG structure impact in guiding placement and routing
  • Automatic Tuning: Framework can be automatically tuned for new accelerator architectures without manual customization
  • Superior Coverage: For 71 combinations of benchmarks and accelerators, LISA can map all of them while ILP cannot map 48 combinations and SA cannot map 21 combinations

Morpher Framework Components

DFG Generation & Data Layout

The framework begins by taking application source code with annotated kernel as input. The DFG generation component analyzes the kernel and extracts a dataflow graph representation, capturing all operations and data dependencies. Simultaneously, the SPM Data Layout component determines optimal data placement in scratchpad memory.

Abstract Architecture Model

Users specify their CGRA architecture using Morpher's architecture description language in Chisel format. This abstract model defines PE capabilities, interconnect topology, memory organization, and other architectural parameters. The architecture interpreter and auto-generator then produce the necessary infrastructure for compilation and simulation.

CGRA Mapper

The core compilation component takes the DFG and architecture model to generate mapping configurations. This involves sophisticated algorithms (PANORAMA for complex kernels, HiMap for regular kernels, LISA for portable low-level mapping) to assign operations to PEs and route data dependencies through the interconnect.

Validation & Verification: Morpher includes both C++ simulation for fast functional verification and Verilog RTL generation for cycle-accurate hardware validation and FPGA emulation

Compilation Performance Analysis

Morpher is enabled by underlying compilation technology that generates quality mapping but at 13× faster compilation time compared to existing approaches. The combination of PANORAMA for complex kernels, HiMap for regular kernels, and LISA for portable low-level mapping provides comprehensive coverage across all application types.

Performance Highlights

  • PANORAMA: Improves throughput of mapped complex loops kernels with large and irregular DFGs by up to 2.6× with 8.7× faster compilation compared to SPR
  • HiMap: Offers 17.3× and 5× improvement in performance and energy efficiency of mappings compared to state-of-the-art, with compilation time less than 15 minutes for 64×64 CGRA while existing approaches take days
  • LISA: Achieves 594× and 17× compilation time reduction compared to ILP and SA used in SPR, successfully mapping all 71 benchmark-accelerator combinations
